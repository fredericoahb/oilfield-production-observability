# Cloud demo deploy (free tiers)

This repo supports a **local-first** Docker Compose stack (Grafana + Prometheus + Tempo).  
If you want a **public cloud demo URL**, this guide deploys:

- **2 Render free web services** (ingestion-service + kpi-service)
- **1 Supabase free Postgres** (persistence)
- **1 Grafana Cloud free stack** (traces + metrics via OTLP)

Why these choices:
- Render has a **Free web service** tier but **spins down after ~15 minutes idle** and has monthly free-hour limits. Render documents these Free service limits.  
- Supabase Free is a real Postgres instance with a small quota, and **free projects may be paused after ~7 days of inactivity** (you can unpause in the dashboard).

> All of the above are optional. You can always run everything 100% local.

---

## 0) Prereqs

- GitHub repo with this code
- Render account
- Supabase account
- Grafana Cloud account/stack

---

## 1) Create the database (Supabase)

1. Create a new Supabase project.
2. Copy connection details:
   - host
   - port
   - database
   - user
   - password

3. Recommended: keep SSL enabled (Supabase uses TLS). In Render we set `DB_SSLMODE=require`.

> Note: Supabase Free projects can be paused after one week of inactivity. If you use this only for a demo, that's fine.

---

## 2) Create a Grafana Cloud stack (OTLP endpoint)

1. Create a Grafana Cloud stack.
2. In the stack, open **OpenTelemetry** and click **Configure**.
3. Grafana Cloud will generate:
   - `OTEL_EXPORTER_OTLP_ENDPOINT`
   - `OTEL_EXPORTER_OTLP_HEADERS`
   - (and sometimes the recommended protocol)

For details, see Grafana Cloud's docs about sending data to the OTLP endpoint.

### Mapping to this repo

You will set in Render:
- `OTEL_EXPORTER_OTLP_ENDPOINT`  → copy from Grafana Cloud
- `OTEL_EXPORTER_OTLP_HEADERS`   → copy from Grafana Cloud
- `METRICS_OTLP_URL`             → your OTLP **metrics** endpoint URL (must end with `/v1/metrics`)
- `METRICS_OTLP_AUTHORIZATION`   → only the Authorization value (e.g. `Basic ...`)

Tip: If Grafana Cloud gives you `OTEL_EXPORTER_OTLP_HEADERS="Authorization=Basic ..."` then set:
- `METRICS_OTLP_AUTHORIZATION` to `Basic ...` (without the `Authorization=` prefix)

---

## 3) Deploy to Render using the Blueprint

This repo includes a **Render Blueprint** file at the repository root: `render.yaml`.

### Steps

1. Push the repo to GitHub.
2. In Render: **New** → **Blueprint** → select your GitHub repo.
3. Render will detect `render.yaml` and create two services.

After the services are created, open each service and set the required env vars (Render will prompt for anything marked `sync: false`).

### Required environment variables

Database (same values in both services):
- `DB_HOST`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- (already defaulted) `DB_PORT=5432`
- (already defaulted) `DB_SSLMODE=require`

Grafana Cloud traces:
- `OTEL_EXPORTER_OTLP_ENDPOINT`
- `OTEL_EXPORTER_OTLP_HEADERS`

Grafana Cloud metrics (Micrometer → OTLP):
- `METRICS_OTLP_URL` (must end with `/v1/metrics`)
- `METRICS_OTLP_AUTHORIZATION` (value only, e.g. `Basic ...`)


## 5) Notes & troubleshooting

### Render free services spin down
Free web services spin down after a short idle period. The first request after idle may take ~1 minute to respond.

### JVM memory
Render Free is small. This repo sets `JAVA_OPTS` in `render.yaml` to keep the JVM within limits.

### If metrics don't show up in Grafana Cloud
- Confirm `METRICS_OTLP_URL` ends with `/v1/metrics`
- Confirm `METRICS_OTLP_AUTHORIZATION` matches the token generated by Grafana Cloud.
- Check Render logs for HTTP 401/403 to the OTLP endpoint.
